# Technical Writing

기술 문서 작성의 전체 프로세스를 3단계로 안내합니다.

## Workflow

```
Step 1: 문서 유형 정하기 → Step 2: 정보 구조 만들기 → Step 3: 문장 다듬기
```

## Instructions

사용자의 문서 작성 상황을 파악한 후, 아래 3단계를 순차적으로 진행합니다.
각 단계가 완료되면 사용자의 확인을 받고 다음 단계로 넘어갑니다.

### Step 1: 문서 유형 정하기

`/determining-document-type` 스킬을 사용하여 문서 유형을 결정합니다.

**수집할 정보:**
- 문서 목표
- 독자 수준
- 프로젝트 상황
- 추가 고려 사항

**결과물:**
- 적합한 문서 유형 (학습 / 문제 해결 / 참조 / 설명)
- 해당 유형에 맞는 작성 가이드

### Step 2: 정보 구조 만들기

`/structuring-documentation` 스킬을 사용하여 문서 구조를 설계합니다.

**수집할 정보:**
- 문서 초안 또는 목차
- (선택) 문서 목표 및 독자
- (선택) 현재 겪고 있는 문제점

**결과물:**
- 개선된 문서 구조
- 제목 및 섹션 구성 제안
- 정보 배치 순서

### Step 3: 문장 다듬기

`/refining-sentences` 스킬을 사용하여 문장을 개선합니다.

**수집할 정보:**
- 개선이 필요한 문장 또는 문단

**결과물:**
- 명확하고 간결한 문장으로 개선된 텍스트
- 적용된 원칙에 대한 설명

## Usage

### 전체 프로세스 진행

```
/technical-writing

문서 목표: React Hook 개념을 설명하는 문서
독자: React 초급 개발자
```

### 특정 단계만 진행

각 단계는 개별 스킬로도 사용할 수 있습니다:

- `/determining-document-type` - 문서 유형만 결정
- `/structuring-documentation` - 구조만 개선
- `/refining-sentences` - 문장만 다듬기

## Notes

- 각 단계는 이전 단계의 결과를 기반으로 진행됩니다.
- 사용자가 특정 단계를 건너뛰고 싶다면 해당 단계를 생략할 수 있습니다.
- 문서 초안이 이미 있는 경우 Step 2 부터 시작할 수 있습니다.
- 구조가 확정된 문서는 Step 3 부터 시작할 수 있습니다.
