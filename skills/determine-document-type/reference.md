# Document Types in Technical Writing

기술 문서 작성 시 독자의 의도에 따라 문서를 분류하는 방법을 설명합니다.

---

## Four Document Types

| 문서 유형 | 목적 | 대상 독자 |
|----------|------|----------|
| Learning | 새로운 기술 학습 | 초심자 |
| Problem-solving | 특정 문제 해결 | 경험 있는 사용자 |
| Reference | 기능/API 조회 | 기초 지식 보유자 |
| Explanation | 개념과 원리 심층 이해 | 깊은 이해가 필요한 독자 |

**Learning-focused documents** - 새로운 기술을 처음 접하는 초심자를 위한 문서. 튜토리얼, 시작 가이드 등이 해당됩니다.

**Problem-solving documents** - 경험 있는 사용자가 특정 문제를 해결할 수 있도록 돕는 문서. 트러블슈팅 가이드 등이 해당됩니다.

**Reference documents** - 기초 지식을 가진 사용자가 특정 기능이나 API를 조회할 수 있는 문서. API 레퍼런스 등이 해당됩니다.

**Explanation documents** - 원리와 배경에 대한 깊은 개념적 이해를 제공하는 문서.

> **Note**: 효과적인 문서는 한 가지 유형에 엄격히 따르기보다 여러 유형을 결합할 수 있습니다.

---

## Learning Documents

학습 문서는 독자가 새로운 기술이나 도구를 이해할 수 있도록 돕습니다. 문서를 완료한 후 무엇을 할 수 있는지 명확히 보여줘야 합니다.

### Three Core Principles

**1. Seamless Progression (막힘 없는 진행)**
- 독자가 오류나 장애물 없이 전체 과정을 완료할 수 있어야 함
- 모든 코드 예제는 테스트되고 검증되어야 함
- 사전 요구사항을 명확히 제시해야 함

**2. Step-by-Step Structure (단계별 구조)**
- 내용을 순차적으로 구성하여 쉽게 따라갈 수 있도록 함
- 예제는 간단한 것에서 복잡한 것으로 점진적 진행

**3. Executable Code Examples (실행 가능한 코드 예제)**
- 개념적 설명만이 아닌 실용적이고 실행 가능한 코드 샘플 포함
- 예제는 핵심 기능을 강조해야 함
- 작동하는 코드가 이해도와 활용성을 크게 향상시킴

### Getting Started vs. Tutorials

| 유형 | 목적 |
|------|------|
| Getting Started | 핵심 개념과 기본 워크플로우 소개, 필수 설정과 전반적 이해에 집중 |
| Tutorials | 구체적인 결과물이 있는 명확한 목표 포함, 구조화된 실행 단계를 통해 개념 습득 |

### Advanced Techniques

**FAQ Sections** - 튜토리얼의 핵심 내용에 집중하면서 일반적인 문제는 마지막에 별도로 다룸

**Collapsible Components** - 보충 세부사항, 트러블슈팅 팁, 고급 개념을 숨겨 주요 내러티브를 간결하게 유지

### Checklist

- [ ] 학습 목표가 명확히 명시되었는가?
- [ ] 따라 하는 중 오류가 없는가?
- [ ] 단계별로 구성되어 있는가?
- [ ] 실행 가능한 코드 예제가 포함되어 있는가?

## Problem-Solving Documents

문제 해결 문서는 독자가 문제를 빠르고 효율적으로 해결할 수 있도록 돕습니다.

### Key Principles

**1. Define the Problem Clearly (문제를 명확히 정의)**
- 근본 원인과 증상을 구분
- 오류 메시지와 로그 예제를 포함하여 독자가 자신의 상황을 인식하도록 도움

**2. Provide Immediately Applicable Solutions (즉시 적용 가능한 해결책 제공)**
- 해결책은 명확하고 실행 가능해야 함
- 코드 예제, 명령어, 설정 단계 포함
- 기본 원리를 설명하여 독자의 이해를 강화

**3. Consider Environmental Variations (환경 변수 고려)**
- 운영 체제와 라이브러리 버전에 따른 차이를 고려
- 문제가 다르게 나타날 수 있는 경우를 다룸

### Document Types

**How-to Guides**
- 단계별 절차를 통해 특정 기능 구현에 집중
- 실행 가능한 코드 예제 포함
- 컨텍스트(왜 중요한지)와 실용적 구현 세부사항 모두 제공

**Troubleshooting Documents**
- 문제가 이미 발생했을 때 진단과 디버깅에 중점
- 기본 진단부터 시작하여 대안적 해결책 제시
- 독자가 해결을 확인할 수 있도록 보장

### Checklist

- [ ] 오류 설명과 함께 기초 지식을 제공하는가?
- [ ] 즉시 구현 가능한 해결책이 포함되어 있는가?
- [ ] 환경별 고려사항을 다루고 있는가?

## Reference Documents

레퍼런스 문서는 사용자가 특정 기능이나 API를 빠르게 찾아볼 수 있도록 합니다.

### Core Principles

**Accuracy and Completeness (정확성과 완전성)**
- 모든 정보는 정확해야 하고, 누락된 부분이 없어야 함

**Consistent Structure (일관된 구조)**
- 표준화된 형식으로 독자가 문서를 탐색하도록 도움
- API의 경우: 함수명 → 매개변수 → 반환값 → 코드 예제 패턴 따름

**Searchable Design (검색 가능한 설계)**
- 목차, 키워드 검색, 앵커 링크를 사용하여 독자가 필요한 정보를 빠르게 찾도록 도움

**Code Examples (코드 예제)**
- 실용적인 예제로 함수나 API 사용 방법을 보여줌
- 개발자에게 구현을 더 명확하게 만듦

### Practical Application

레퍼런스 문서 구성 예시:

```
1. 간단한 기능 개요 (가치 설명)
2. 함수 시그니처 (빠른 이해용)
3. 상세한 매개변수 및 반환값 문서
4. 여러 코드 예제 (기본, 고급 패턴)
```

### Advanced Recommendations

- 광범위한 레퍼런스 자료는 주제별로 구성하고 사이드 내비게이션 메뉴 활용
- 중요한 설정 정보(API 키, 인증, 헤더)를 문서 시작 부분에 눈에 띄게 배치

### Checklist

- [ ] 검색 가능한 구조로 되어 있는가?
- [ ] 정확하고 완전한 정보를 포함하는가?
- [ ] 일관된 형식을 따르는가?
- [ ] 실용적인 코드 예제가 있는가?

## Explanation Documents

설명 문서는 독자가 기술 개념과 원리에 대한 포괄적인 이해를 발전시킬 수 있도록 돕습니다.

### Key Principles

**1. Provide Background and Context (배경과 컨텍스트 제공)**
- 기술이 왜 등장했고 어떤 문제를 해결하는지 설명
- 독자가 기술 선택의 이유를 이해하도록 도움

**2. Use Visual Materials Actively (시각 자료 적극 활용)**
- 복잡한 개념은 다이어그램, 플로우차트, 테이블로 설명
- 시스템 구조, 데이터 흐름, 컴포넌트 관계를 직관적으로 보여줌

### Document Examples

**Concept Understanding (개념 이해)**

기술적 메커니즘을 설명하는 방법:
1. 기존 방식의 근본적인 문제 확립
2. 핵심 개념과 장점 정의
3. 운영 단계를 논리적으로 설명
4. 프로세스 흐름을 보여주는 시각적 다이어그램으로 보충

**Domain Knowledge (도메인 지식)**

도메인 이해를 구축하는 방법:
1. 핵심 요소 정의 (예: 상품, 주문, 결제, 배송, 소비자, 판매자)
2. 사용자 행동 흐름을 프로세스를 통해 매핑
3. 개발자가 처리해야 할 잠재적 예외 식별
4. 완전한 도메인 관계를 시각적으로 제시

### Checklist

- [ ] 핵심 원리와 배경이 명확히 설명되었는가?
- [ ] 복잡한 개념이 효과적으로 시각화되었는가?
- [ ] 사전 지식이 적절히 소개되었는가?

## Templates

각 문서 유형별 템플릿:

### Learning Document Template

```markdown
# [제목]

## 학습 목표
이 문서를 완료하면 다음을 할 수 있습니다:
- 목표 1
- 목표 2

## 사전 요구사항
- 요구사항 1
- 요구사항 2

## Step 1: [단계 제목]
[설명 및 코드 예제]

## Step 2: [단계 제목]
[설명 및 코드 예제]

## 다음 단계
[추가 학습 자료 링크]
```

### Problem-Solving Document Template

```markdown
# [문제 제목]

## 문제 정의
[증상과 오류 메시지]

## 원인
[근본 원인 설명]

## 해결 방법

### 방법 1: [제목]
[단계별 해결책]

### 방법 2: [제목]
[대안적 해결책]

## 확인
[해결 여부 확인 방법]
```

### Reference Document Template

```markdown
# [API/기능명]

## 개요
[간단한 설명]

## 시그니처
```code
[함수 시그니처]
```

## 매개변수
| 이름 | 타입 | 설명 |
|------|------|------|
| param1 | string | 설명 |

## 반환값
[반환값 설명]

## 예제
[코드 예제들]
```

### Explanation Document Template

```markdown
# [개념 제목]

## 배경
[왜 이 개념이 필요한지]

## 핵심 개념
[개념 설명]

## 작동 원리
[상세 메커니즘]

## 시각 자료
[다이어그램, 플로우차트]

## 관련 개념
[연결된 주제들]
```
