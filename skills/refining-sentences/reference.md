# Sentence Writing Guide

기술 문서에서 명확하고 이해하기 쉬운 문장을 작성하는 핵심 원칙을 다룹니다.

---

## Key Principles Overview

| 원칙 | 설명 |
|------|------|
| Clarify Subjects | 도구나 시스템이 아닌 개발자를 주어로 |
| Keep Essential | 짧고 간결한 문장 유지 |
| Write Concretely | 명사보다 동사, 모호함보다 정확함 |
| Natural Korean | 불필요한 한자어 제거, 번역체 교정 |
| Consistency | 공식 용어 사용, 동일 개념 동일 표현 |

---

## Clarify Sentence Subjects

기술 문서의 독자는 문서에 설명된 작업을 수행하는 개발자입니다. 따라서 개발자가 문장의 암묵적 또는 명시적 주어가 되어야 더 명확하고 자연스러운 문장이 됩니다.

### Avoid Using Tools as the Subject

도구나 기술 시스템에 행위를 귀속시키기보다 개발자가 할 수 있는 것으로 문장을 구성합니다.

| Bad | Good |
|-----|------|
| 이 라이브러리는 데이터베이스 초기화를 수행합니다 | 이 명령어를 실행하면 데이터베이스를 초기화할 수 있습니다 |
| 이 코드는 인증 토큰을 추가합니다 | 이 코드로 인증 토큰을 추가하세요 |

> **Exception**: 도구가 실제로 어떻게 작동하는지 설명할 때(개발자가 무엇을 하는지가 아닌)는 도구가 주어가 될 수 있습니다.

### Use Active Voice

수동태는 주어를 불분명하게 하고 문장을 복잡하게 만듭니다. 능동태는 기술 문서를 더 직관적이고 이해하기 쉽게 만듭니다.

| Bad (수동태) | Good (능동태) |
|-------------|--------------|
| 애플리케이션이 실행되기 전에 설정이 완료되어야 합니다 | 애플리케이션을 실행하기 전에 설정을 완료하세요 |

## Keep Information Essential

> 짧은 문장은 독자가 내용을 이해하고 정보를 처리하기 더 쉽습니다.

긴 문장은 핵심 포인트를 가리고 오해를 일으킬 위험이 있습니다.

### Keep Sentences Brief and Focused

각 문장을 하나의 아이디어로 제한합니다.

| Bad | Good |
|-----|------|
| 버튼을 클릭하면 다음 단계로 이동하게 되며, 그 이후의 작업을 진행할 수 있습니다 | 버튼을 클릭하면 다음 단계로 이동합니다. 이동 후 이어서 작업을 진행하세요. |

### Minimize Metadiscourse

메타담론은 메시지를 직접 전달하기보다 지원하는 문구를 말합니다. 이는 정보 밀도를 줄이고 이해 속도를 늦춥니다.

**피해야 할 표현:**
- "앞에서 설명했지만"
- "아시다시피"
- "다시 말해서"
- "결론적으로 말하자면"

## Write Concretely

기술 문서는 독자가 필요한 정보를 빠르게 찾고 적용할 수 있어야 합니다.

### Use Verbs Instead of Nouns

직접적인 동사가 명사화된 표현보다 행동을 더 명확히 전달합니다.

| Bad | Good |
|-----|------|
| 코드 최적화 진행 후 배포 수행이 필요합니다 | 코드를 최적화한 후 배포하세요 |

### Replace Ambiguous Phrasing with Precision

"가능성이 있다", "일부 경우" 같은 표현은 불확실성을 만듭니다. 정확한 조건을 명시하세요.

| Bad | Good |
|-----|------|
| 일부 브라우저에서 문제가 발생할 수 있습니다 | Safari 15 이하에서 레이아웃이 깨질 수 있습니다 |

### Include Complete Information

문장이 누가, 무엇을, 어디서, 어떻게에 답하는지 확인합니다.

| Bad | Good |
|-----|------|
| 자동으로 저장됩니다 | 서버에 자동으로 저장됩니다 |

### Provide Concrete Metrics

"많을 때", "빠르게" 같은 상대적 용어를 구체적인 숫자로 대체합니다.

| Bad | Good |
|-----|------|
| 데이터가 많을 때 응답이 느려집니다 | 10,000건을 넘으면 응답 시간이 1초 이상 걸립니다 |

### Explain Referenced Context

다른 섹션의 정보를 참조할 때 개념이 어디서 소개되었고 나중에 어떻게 사용될지 표시합니다.

## Natural Korean Expression

기술 문서에서 더 자연스럽고 간결한 한국어를 작성하는 방법입니다.

### Remove Unnecessary Chinese-Origin Words

"수행하다", "실행하다", "진행하다" 같은 단어는 의미 손실 없이 제거할 수 있는 경우가 많습니다.

| Bad | Good |
|-----|------|
| 로그 파일을 삭제하는 작업을 수행합니다 | 로그 파일을 삭제합니다 |
| 설정 변경을 수행한 후 적용을 진행합니다 | 설정을 바꾼 뒤 적용합니다 |

### Fix Translation-Style Expressions

영어는 자연스럽게 동사를 명사 형태(-tion, -ing, -ment)로 변환하지만, 한국어는 동사를 직접 사용할 때 더 자연스럽게 읽힙니다.

| Bad (번역체) | Good (자연스러운 한국어) |
|-------------|----------------------|
| API 키를 이용한 사용자 인증 처리가 완료된 후... | API 키로 사용자를 인증한 후... |
| 이 API를 통해 | 이 API로 |
| ~를 통한 | ~로 / ~으로 |

## Consistency

용어와 표현의 일관성은 독자의 이해를 높이고 신뢰도를 유지하는 데 필수적입니다. 같은 개념을 다양하게 표현하면 혼란을 초래하고 검색 편의성을 해칩니다.

### Follow Official Technical Terminology

위키피디아와 공식 문서의 표기를 따르며, 대소문자에 주의합니다.

| Bad | Good |
|-----|------|
| K8 | 쿠버네티스(Kubernetes) |
| 자바스크립트 비동기 실행 | JavaScript의 비동기 실행 |

### Avoid Multiple Expressions for Same Concept

같은 의미는 동일한 용어로 표현합니다.

| Bad | Good |
|-----|------|
| 파일 추가 / 파일 첨부 / 파일 업로드 혼용 | "파일 업로드"로 통일 |

### Spell Out Abbreviations Before Use

> 약어는 첫 등장 시 전체 이름을 괄호에 표기합니다.

| Bad | Good |
|-----|------|
| CSR은... | 클라이언트 사이드 렌더링(Client-Side Rendering, CSR)은... |
| SSR을 사용하면... | 서버 사이드 렌더링(Server-Side Rendering, SSR)을 사용하면... |

### Consider Frequency for Foreign Words

정확한 맞춤법보다 업계 관례와 독자 친숙도를 우선합니다. 구글 트렌드에서 5배 이상 사용되는 표기를 선호합니다.

## Checklist Summary

### Subject Clarity
- [ ] 개발자가 문장의 주어인가?
- [ ] 능동태를 사용하는가?

### Essential Information
- [ ] 각 문장이 하나의 아이디어만 담는가?
- [ ] 불필요한 메타담론이 제거되었는가?

### Concreteness
- [ ] 명사 대신 동사를 사용하는가?
- [ ] 모호한 표현 대신 정확한 조건을 명시하는가?
- [ ] 구체적인 수치를 제공하는가?

### Natural Korean
- [ ] 불필요한 한자어가 제거되었는가?
- [ ] 번역체 표현이 교정되었는가?

### Consistency
- [ ] 공식 기술 용어를 사용하는가?
- [ ] 같은 개념에 동일한 표현을 사용하는가?
- [ ] 약어의 첫 등장 시 전체 이름을 표기하는가?

## Practice Examples

### Before & After

**주어 명확히 하기:**
```
Before: 이 라이브러리는 데이터베이스 초기화를 수행합니다
After:  이 명령어를 실행하면 데이터베이스를 초기화할 수 있습니다
```

**간결하게 쓰기:**
```
Before: 버튼을 클릭하면 다음 단계로 이동하게 되며, 그 이후의 작업을 진행할 수 있습니다
After:  버튼을 클릭하면 다음 단계로 이동합니다. 이동 후 이어서 작업을 진행하세요.
```

**구체적으로 쓰기:**
```
Before: 데이터가 많을 때 응답이 느려집니다
After:  10,000건을 넘으면 응답 시간이 1초 이상 걸립니다
```

**자연스러운 한국어:**
```
Before: 설정 변경을 수행한 후 적용을 진행합니다
After:  설정을 바꾼 뒤 적용합니다
```

**일관성 유지:**
```
Before: CSR은...
After:  클라이언트 사이드 렌더링(Client-Side Rendering, CSR)은...
```
